```cmake
# CMakeLists.txt
configure_file(
  "${CMAKE_CURRENT_SOURCE_DIR}/IccAnalyzerVersion.h.in"
  "${CMAKE_CURRENT_BINARY_DIR}/IccAnalyzerVersion.h"
)

# Then in .cpp files:
#include "IccAnalyzerVersion.h"
printf(ICCANALYZER_LITE_VERSION_DISPLAY "\n");
```

### Benefits
- Single point of version maintenance
- Eliminates duplicate string literals
- Reduces merge conflicts during version bumps
- Can be extracted by build tools

## 6. INCLUDE GUARD CONSISTENCY

### Current State: All #ifndef (No #pragma once)

**Header file count**: 29 total
- **0 files** use `#pragma once`
- **29 files** use `#ifndef` pattern

### Guard Style Inconsistency

Two naming conventions observed:

#### Convention 1: _ICCANALYZER* (18 files) — Leading underscore
```cpp
// IccAnalyzerValidation.h
#ifndef _ICCANALYZERVALIDATION_H
#define _ICCANALYZERVALIDATION_H

// IccAnalyzerXMLExport.h
#ifndef _ICCANALYZERXMLEXPORT_H
#define _ICCANALYZERXMLEXPORT_H
```

#### Convention 2: ICC_HEURISTICS_* (11 files) — No leading underscore, snake_case
```cpp
// IccHeuristicsHelpers.h
#ifndef ICC_HEURISTICS_HELPERS_H
#define ICC_HEURISTICS_HELPERS_H

// IccHeuristicsTagValidation.h
#ifndef ICC_HEURISTICS_TAG_VALIDATION_H
#define ICC_HEURISTICS_TAG_VALIDATION_H
```

### Issues
1. **Reserved names**: Leading underscore followed by capital letter (`_ICC`) is technically reserved per C++ standard
2. **Inconsistent style**: Makes codebase look unmaintained
3. **No modern pragmas**: `#pragma once` cleaner and widely supported

### Recommended Standardization

**Preferred: Convention 2 (ICC_* without leading underscore, UPPER_SNAKE_CASE)**

```cpp
// Standard pattern for all headers:
#ifndef ICC_ANALYZER_VALIDATION_H
#define ICC_ANALYZER_VALIDATION_H
// ... content ...
#endif // ICC_ANALYZER_VALIDATION_H
```
